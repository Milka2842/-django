Проект представляет собой веб-сайт на Django для просмотра аниме. Он включает главную страницу с карточками аниме, страницу просмотра с описанием, постером, скриншотами и рейтингом, а также админ-панель для управления контентом. Проект адаптирован для локального запуска и тестирования, использует SQLite вместо PostgreSQL, а все внешние сервисы (Celery, Redis, индексация Google) отключены для упрощения.

Для запуска в GitHub Codespaces открой репозиторий, нажми зелёную кнопку Code, выбери вкладку Codespaces и создай новый кодспейс. После загрузки окружения открой терминал и выполни последовательно несколько команд. Сначала создай виртуальное окружение командой python -m venv venv, затем активируй его командой source venv/bin/activate (для Linux) или venv\Scripts\activate (для Windows). После активации установи все необходимые зависимости командой pip install -r requirements.txt. Затем примени миграции для создания базы данных командой python manage.py migrate. После этого создай суперпользователя для входа в админку командой python manage.py createsuperuser – укажи логин, почту и пароль. Наконец, запусти сервер командой python manage.py runserver. В терминале появится ссылка на локальный адрес, по которому можно открыть сайт в браузере.

В проекте уже добавлено одно тестовое аниме «Человек-бензопила» с описанием, постером и скриншотами, поэтому главная страница не будет пустой. Для добавления новых аниме можно использовать админ-панель по адресу /admin, предварительно войдя под созданным суперпользователем. Также можно добавлять записи через Django shell, но через админку проще.

Все настройки проекта адаптированы для SQLite – база данных создаётся в файле db.sqlite3 в корне проекта. Кеширование отключено, Celery и Redis не требуются. Для корректной работы страницы аниме в SQLite добавлена пользовательская функция jsonb_array_length, которая эмулирует поведение PostgreSQL. Эта функция зарегистрирована в настройках и не требует дополнительных действий.
